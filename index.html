Arquitetura de Rede e Acesso Externo

Para garantir a acessibilidade global, foi desenhada uma arquitetura de rede que expõe os serviços da máquina virtual de forma segura e eficiente.

O Caminho da Conexão

Um pedido externo, como o acesso a esta página, atravessa várias camadas. O router, atuando como porteiro, utiliza regras de DMZ e Port Forwarding para encaminhar o tráfego corretamente para o servidor Ubuntu, que está isolado numa Máquina Virtual.

  • DMZ (Demilitarized Zone): Atua como um funil, direcionando tráfego não especificado para o nosso servidor.
  • Port Forwarding: Cria "atalhos" diretos para serviços específicos como SSH (porta 22) e HTTP (porta 80).

A Estrela do Projeto: NexoCli

Implementámos o NexoCli, um fork do `gemini-cli` da Google, como uma interface de gestão inteligente e conversacional.

Instalação e Demonstração

Após a instalação de dependências como Node.js, o NexoCli é instalado via NPM. A sua capacidade de interagir com o sistema e de utilizar ferramentas como a Pesquisa Google para responder a questões complexas é o seu maior trunfo.

A animação ao lado simula uma interação real, demonstrando como o NexoCli processa um pedido e devolve uma resposta estruturada.

$ nexo-cli ask "explain protein sequencing"
